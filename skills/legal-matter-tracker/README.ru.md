# Legal Matter Tracker

Сканирует файлы воркспейса по имени клиента или названию дела и строит полную хронологию событий — без интеграций, без настройки, только из ваших локальных заметок.

---

## Обзор

Legal Matter Tracker сканирует ваш воркспейс Cowork в поиске файлов, связанных с конкретным клиентом или делом, извлекает датированные события и ключевые факты, и собирает из них структурированную хронологию. Укажите имя клиента или идентификатор дела — скил найдёт нужные заметки, договоры и документы и выстроит полную историю в хронологическом порядке, со ссылками на источники для каждой записи.

Никаких CRM, никаких систем управления делами, никаких API-ключей. Работает исключительно с файлами `.md` и `.txt`, которые уже есть в вашем воркспейсе. Незаменим перед встречей с клиентом, при подготовке статус-обновления, в ходе due diligence или когда нужно восстановить историю проекта из разрозненных заметок.

---

## Требования

- Файлы воркспейса, связанные с клиентом или делом (`.md` или `.txt`)
- Файлы должны содержать имя клиента/дела в названии файла или в заголовках/первых строках
- Внешние инструменты и интеграции не нужны

**Совет:** Скил работает лучше, когда заметки названы по имени клиента или дела (например, `ромашка-заметки.md`, `дело-смирнов.md`). При этом он также ищет по содержимому файлов, поэтому файлы с упоминанием клиента в заголовках тоже будут найдены.

---

## Как использовать

1. **Запустите скил**, указав имя клиента или дела
   - Напишите: «хронология по делу Ромашка» или «отчёт по клиенту Иванов»
   - Скил автоматически просканирует воркспейс

2. **Изучите хронологию**
   - События отсортированы по дате с указанием источника
   - Записи без дат сгруппированы в отдельный раздел
   - Сводка ключевых фактов: стороны, статус, открытые вопросы

3. **Сохраните при необходимости**
   - Напишите «сохрани» — скил запишет отчёт в `output/matter-timeline-[клиент]-YYYY-MM-DD.md`

4. **Сузьте область поиска** (опционально)
   - Укажите папку для ограничения поиска: «хронология по делу Ромашка в папке `clients/`»

## Примеры

### Пример 1: Хронология по клиенту

**Запуск:** «хронология по делу Ромашка»

**Результат:**
```
# Matter Timeline — Ромашка
**Дата:** 2026-04-29
**Файлов просканировано:** 12  |  **Совпало:** 3  |  **Извлечено событий:** 7

---

## Хронология событий

| Дата | Событие | Источник |
|------|---------|----------|
| 2025-01-15 | Первичная консультация. Клиент представил спор об ИС. | `notes/ромашка-заметки.md` |
| 2025-02-03 | Договор подписан. Вознаграждение: 1 200 000 ₽. | `contracts/ромашка-договор.md` |
| 2025-03-10 | Стадия discovery завершена. Выявлено 3 ключевых документа. | `notes/ромашка-заметки.md` |

---

## Сводка ключевых фактов
- **Стороны:** Ромашка (клиент), Smith & Partners LLP (сторона ответчика)
- **Статус дела:** Открыто (последняя активность: 2025-03-10)
- **Открытые вопросы:** Документы по discovery ожидают проверки
```

---

### Пример 2: Поиск по папке

**Запуск:** «отчёт по делу Альфа в папке `cases/`»

Скил сканирует только папку `cases/` в поиске файлов с упоминанием «Альфа» и возвращает хронологию в ограниченной области.

## Триггеры

Используйте любую из этих фраз для запуска скила:

| Русский | English |
|---------|---------|
| хронология дела [название] | case timeline for [name] |
| трекер дела | matter tracker |
| что было по клиенту [имя] | track [client name] |
| история клиента [имя] | client history for [name] |
| отчёт по делу | matter report |
| хронология событий по [клиент] | summarize [client] matter |
