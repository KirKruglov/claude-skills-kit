# Legal Matter Tracker — Руководство пользователя

Практические сценарии использования Legal Matter Tracker.

---

## Быстрый старт

1. Напишите: «хронология дела [имя клиента или дела]»
2. Скил сканирует воркспейс и строит хронологию из найденных файлов
3. Напишите «сохрани» — отчёт сохранится в `output/matter-timeline-[клиент]-YYYY-MM-DD.md`

---

## Сценарии

### Сценарий 1: Подготовка к встрече с клиентом

**Контекст:** До звонка с клиентом 20 минут. Заметки разбросаны по нескольким файлам — записи встреч, резюме переписки, папка с договорами.

**Что делать:**
1. Откройте сессию Cowork с выбранной папкой проекта
2. Напишите: «хронология дела Ромашка»
3. Скил просканирует все подходящие файлы и вернёт единый хронологический вид всех ключевых событий

**Ожидаемый результат:** Хронология с датированными записями из всех найденных файлов — даты договоров, ключевые решения, открытые вопросы. Вы входите на звонок с полным контекстом.

**Совет:** Называйте файлы заметок по имени клиента (`ромашка-заметки.md`, `ромашка-встреча-2025-01.md`) — скил сначала ищет по имени файла, это быстрее и точнее.

---

### Сценарий 2: Восстановление истории дела после перерыва

**Контекст:** Вернулись из отпуска и нужно разобраться с делом, которое продвигалось без вас.

**Что делать:**
1. Напишите: «история клиента Иванов»
2. Скил просканирует воркспейс и вернёт все найденные события, отсортированные по дате

**Ожидаемый результат:** Полная хронологическая картина происходившего со ссылками на источники — можно открыть конкретный файл для подробностей.

**Совет:** Если в хронологии не хватает событий, которые точно есть — проверьте, что в нужных файлах имя клиента присутствует в заголовке или в первых 10 строках.

---

### Сценарий 3: Подготовка статус-обновления для клиента

**Контекст:** Клиент запросил сводку по прогрессу. Нужна хронология как основа для отчёта.

**Что делать:**
1. Напишите: «отчёт по делу Альфа — сохрани»
2. Скил строит хронологию и сохраняет её в `output/matter-timeline-alfa-YYYY-MM-DD.md`
3. Откройте файл, отредактируйте язык и отправьте клиенту

**Ожидаемый результат:** Структурированный отчёт, готовый к редактированию — отсортированные события, сводка ключевых фактов, ссылки на источники, которые можно убрать перед отправкой.

---

### Сценарий 4: Поиск в конкретной папке

**Контекст:** Несколько клиентов с похожими именами, или нужно ограничить поиск одной папкой.

**Что делать:**
1. Напишите: «хронология дела Альфа в папке `cases/alpha/`»
2. Скил сканирует только указанную папку

**Ожидаемый результат:** Хронология в рамках заданной области с пометкой в заголовке отчёта о сужении поиска.

---

### Сценарий 5: Противоречивые данные в разных файлах

**Контекст:** В двух файлах разные версии одного факта (например, разные суммы договора).

**Что делать:**
1. Запустите стандартную команду для хронологии дела
2. Скил отметит противоречие маркером `⚠ Conflict` в таблице

**Ожидаемый результат:** Обе версии появятся рядом в хронологии со ссылками на источники — вы сможете определить, какой файл является авторитетным.

---

## Советы

- **Называйте файлы по клиентам или делам:** `ромашка.md`, `дело-смирнов.md` — скил сначала ищет по имени файла.
- **Используйте заголовки:** Начинайте файлы с `# Имя клиента` или `## Проект Альфа` — скил читает заголовки для подтверждения релевантности.
- **Записывайте даты явно:** Включайте даты в заметки (`2025-03-10:`, `10 марта 2025`) — скил извлекает их для хронологической сортировки.
- **Добавляйте задачи inline:** Строки с `- [ ]`, `TODO:` или `?` извлекаются как открытые вопросы в разделе «Ключевые факты».
- **Сохраняйте каждый отчёт:** Команда «сохрани» фиксирует версию хронологии — отчёты накапливаются и дают историю версий.
- **Несколько дел одновременно:** Запускайте скил отдельно для каждого клиента/дела — каждый запуск даёт одну сфокусированную хронологию.
