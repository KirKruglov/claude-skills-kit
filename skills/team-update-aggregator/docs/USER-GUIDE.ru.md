# Агрегатор обновлений команды — Руководство пользователя

## Что делает этот скил

Читает файлы с еженедельными обновлениями членов команды и создаёт отчёт `team-status-YYYY-MM-DD.md`, организованный по людям. Каждый сотрудник получает раздел с:
- Что сделано на этой неделе
- Планы на следующую неделю
- Что мешает работе
- Нагрузка / отпуск / больничный
- Флаги внимания менеджера (⚠️)

---

## Подготовка входных файлов

### Вариант А: Один файл на человека

Создайте по одному файлу `.md` или `.txt` на каждого члена команды. Имя файла становится именем человека в отчёте.

```
/updates/
├── anna-ivanova.md
├── ivan-petrov.md
├── maria-sokolova.md
└── alex-novikov.md
```

Файл может быть произвольным — скил сам извлекает структурированный контент.

**Минимальный пример файла (`anna-ivanova.md`):**
```
Сделала: выпустила поиск, починила 3 баги
На следующей неделе: начну рефакторинг API
Блокер: жду ревью дизайна от Марии
```

### Вариант Б: Один файл с обновлениями всей команды

Используйте заголовки H2 (`##`) или H3 (`###`) с именами сотрудников:

```markdown
## Анна Иванова

Выпустила функцию поиска. Жду ревью дизайна перед началом рефакторинга API.

## Иван Петров

Завершил спринт-планирование и разблокировал data pipeline. На следующей неделе: дашборд отчётности.
```

### Вариант В: Поле с именем в начале файла

Файлы с полем `Name:` или `Кто:` в начале:

```
Кто: Мария Соколова
---
На этой неделе завершила онбординг-флоу. Чувствую небольшую перегрузку — хотелось бы обсудить приоритеты.
```

## Фразы-триггеры

Напишите одну из этих фраз, чтобы запустить скил:

**Русский:**
- `Агрегируй обновления команды из /путь/к/папке/`
- `Составь статус команды из этих файлов: file1.md, file2.md`
- `Скомпилируй people-отчёт по команде`
- `Что делает команда — собери из папки updates/`

**English:**
- `Aggregate team updates from /path/to/folder/`
- `Build team status report for this week`
- `Compile team status from these files`

## Как читать отчёт

Файл `team-status-YYYY-MM-DD.md` состоит из четырёх разделов:

### 1. Summary
2–4 предложения: общее состояние команды, ключевые результаты, главные блокеры, кто требует вашего внимания.

### 2. Team Members
По одному подразделу на каждого человека. Флаги менеджера (⚠️) появляются, когда скил обнаруживает сигналы стресса, перегрузки, нерешённых блокеров или просьб о помощи.

### 3. Cross-Team
- **Общие блокеры** — один и тот же блокер упомянут у 2+ человек
- **Зависимости** — Человек А ждёт от Человека Б
- **Action items без owner** — задачи без назначенного ответственного

### 4. Files Processed
Что включено, пропущено или имело неопознанного владельца.

## Советы

- **Называйте файлы понятно.** `anna.md` работает лучше, чем `update3.md`.
- **Поощряйте единый формат.** Даже 3-строчное обновление (сделал / планирую / мешает) достаточно для скила.
- **Запускайте еженедельно.** Сочетайте с `one-to-one-prep` перед встречами 1-on-1.
- **Большие команды (10+).** Скил добавит пометку «Large team» в начало отчёта — проверьте полноту.

---

## Решение проблем

| Проблема | Решение |
|---------|---------|
| «No files found» | Проверьте путь к папке; убедитесь, что файлы — `.md` или `.txt` |
| Человек отображается как «Unknown» | Добавьте имя как заголовок H1 или поле `Name:` / `Кто:` в файл |
| Файл с отчётом уже существует | Скил спросит перед перезаписью — подтвердите или выберите `-v2` |
| Файл пустой | Скил пропустит его и укажет в «Files with no updates» |
