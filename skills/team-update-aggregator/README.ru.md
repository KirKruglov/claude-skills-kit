# Агрегатор обновлений команды

Скил для Claude, который компилирует еженедельные обновления от членов команды в структурированный **people-centric** статус-отчёт — организованный по людям, не по проектам. Каждый сотрудник получает отдельный раздел с прогрессом, планами, блокерами, нагрузкой и флагами внимания менеджера.

## Обзор

Большинство инструментов для еженедельных дайджестов организуют информацию по проектам или темам. Этот скил организует её **по людям** — чтобы вы видели полную картину по каждому сотруднику с первого взгляда: что сделано, что мешает, насколько человек загружен и требует ли что-то вашего внимания.

Создан для менеджеров, готовящихся к 1-on-1, еженедельному review команды или ведению people-трекера.

## Требования

Локальные файлы `.md` или `.txt` с обновлениями членов команды. Один файл на человека или один файл с несколькими разделами (один на человека).

## Как использовать

1. Укажите Claude путь к папке или список файлов с обновлениями команды
2. Скил определяет каждого сотрудника по имени файла или заголовку внутри него
3. Извлекает прогресс, планы, блокеры, нагрузку и флаги менеджера
4. Результат: файл `team-status-YYYY-MM-DD.md` в рабочей директории

## Примеры

**Пример 1 — Папка с отдельными файлами**
> «Агрегируй обновления команды из /updates/week-17/»

Каждый файл (например, `anna.md`, `ivan.md`) становится отдельным разделом в отчёте.

**Пример 2 — Один файл с несколькими разделами**
> «Составь статус команды из team-updates.md»

Скил разбивает файл по H2/H3-заголовкам с именами и создаёт раздел на каждого человека.

**Пример 3 — Подготовка к 1-on-1**
> «Скомпилируй обновления команды — у меня завтра 1-on-1»

Результат: отчёт с флагами, кто требует внимания.

## Структура отчёта

```
team-status-YYYY-MM-DD.md
├── Summary (состояние команды, ключевые результаты, флаги)
├── Team Members (раздел на каждого сотрудника)
│   ├── Прогресс
│   ├── Планы на следующую неделю
│   ├── Блокеры
│   ├── Нагрузка
│   └── Флаги менеджера ⚠️
└── Cross-Team (общие блокеры, зависимости, action items без owner)
```

## Чем отличается от weekly-digest-synthesizer

| weekly-digest-synthesizer | team-update-aggregator |
|--------------------------|----------------------|
| Организован по **проектам / темам** | Организован по **людям** |
| «Что происходит на проекте Альфа?» | «Как дела у Анны на этой неделе?» |
| Подходит для проектных статус-встреч | Подходит для 1-on-1 и управления людьми |

## Связанные скилы

- **weekly-digest-synthesizer** — дайджест по проектам из нескольких файлов
- **one-to-one-prep** — структурированный prep-документ для индивидуальных 1-on-1
- **hiring-pipeline-reviewer** — еженедельный статус кандидатов из заметок
