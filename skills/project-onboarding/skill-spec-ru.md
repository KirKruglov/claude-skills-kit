# Skill: project-onboarding

Полный onboarding нового проекта в Claude Cowork за одну сессию.

---

## Режимы

| Режим | Триггеры | Действия |
|-------|----------|----------|
| `new` | «создай проект», «onboarding», «новый проект», «настрой проект» | Полный опрос → context.md + folder-instructions.md + file-map + промпты |
| `quick` | «быстрый контекст», «создай контекст проекта» | Блоки 1–3 + автоскан → context.md |
| `scan` | «обнови карту файлов», «пересканируй» | Только модуль File Map → обновление секции в context.md |

---

## Инструкция

### Шаг 1 — Определи режим
Если триггер соответствует `scan` → сразу к модулю File Map.
Если `quick` → интервью блоки 1–3, затем File Map.
Иначе → `new`, все шаги последовательно.

### Шаг 2 — Интервью (режимы new / quick)

Задавай вопросы блоками. После каждого блока жди ответ. По-русски, кратко.

#### Блок 1 — Основное
```
1. Название проекта?
2. Тип: аналитика / контент / бизнес / личный / другой?
3. Статус: Active / On hold / Завершён?
4. Дата старта? Дедлайн?
```

#### Блок 2 — Суть проекта
```
5. Суть проекта в 2–4 предложениях.
6. Vision — какой результат = успех?
```

#### Блок 3 — Текущий этап
```
7. Что делаем прямо сейчас?
8. Конкретный результат этапа?
```

*→ Режим `quick` останавливается здесь. Переходи к модулю File Map.*

#### Блок 4 — Структура
```
9. Ключевые элементы / компоненты / слои проекта?
```

#### Блок 5 — Участники
```
10. Стейкхолдеры, партнёры, внешние зависимости?
```

#### Блок 6 — Ограничения
```
11. Ограничения: время, ресурсы, риски?
```

#### Блок 7 — Открытые вопросы
```
12. Известные нерешённые задачи или вопросы?
```

#### Блок 8 — Дополнительно (опционально)
```
13. Нужны доп. разделы: контекст рынка / методология / roadmap?
```
Если да — 1–2 уточняющих вопроса на каждый выбранный.

### Шаг 3 — Модуль File Map

Определи папку проекта:
- Если подключена папка пользователя (mounted workspace) — используй её корень.
- Если путь был упомянут в интервью — используй его.
- Иначе — спроси: «Укажи путь к папке проекта».

Выполни:
```bash
find [project-folder] -type f \
  -not -path '*/.git/*' \
  -not -path '*/node_modules/*' \
  -not -path '*/__pycache__/*' \
  -not -name '.DS_Store' \
  | head -200
```

Для каждого файла определи: путь, расширение, размер.
Для текстовых (.md, .txt, .csv) — прочти первую строку как описание.
Для остальных — описание `[авто]` по имени файла или `[уточнить]`.

Сгруппируй по папкам. Сформируй секцию по формату:

```markdown
# Структура данных

## [имя папки]/
| Файл | Тип | Размер | Описание |
|------|-----|--------|----------|
| filename.ext | EXT | X KB | описание |
```

В режиме `scan` — найди существующий `context.md`, замени секцию «Структура данных». Остальное не трогай.

### Шаг 4 — Генерация context.md (режимы new / quick)

Правила:
- Не вставляй ответы дословно — пиши связный структурированный текст
- Сохраняй терминологию пользователя
- Не домысливай — только факты из ответов
- Недостающее → `[уточнить]`
- Секцию «Структура данных» бери из модуля File Map

Шаблон: см. `resources/context-template.md`

Выход: `context.md`

### Шаг 5 — Модуль Rules (только режим new)

На основе типа проекта (ответ 2) загрузи шаблон из `resources/rules-templates.md`.
Добавь общие правила (file discovery, язык, поведение).
Адаптируй под ответы пользователя.

Выход: `folder-instructions.md`

После генерации спроси: «Правила готовы. Нужно отредактировать или добавить свои?»

### Шаг 6 — Модуль Starter Prompts (только режим new)

Спроси: «Создать стартовые шаблоны промптов для этого типа проекта?»
Если да — загрузи шаблоны из `resources/prompt-templates.md` по типу проекта.
Адаптируй под контекст (название проекта, терминология).

Выход: файлы в `resources/prompts/`

### Шаг 7 — Итог

Покажи таблицу:
```
| Файл | Статус |
|------|--------|
| context.md | Создан, X блоков заполнено, Y [уточнить] |
| folder-instructions.md | Создан / Пропущен |
| resources/prompts/*.md | Создано N шаблонов / Пропущено |
```

Предоставь все файлы для скачивания.

---

## Ограничения

- Максимум 200 файлов в автоскане (больше → предупреди и покажи top-level)
- SKILL.md не генерирует prompt-builder промпты — это отдельный скилл
- Не изменяй файлы проекта кроме context.md (в режиме scan)
