# Cowork Session Planner

Начинайте каждую Cowork-сессию с чёткого фокуса — статус проекта, цель и план работы в одном брифе.

---

## Обзор

Cowork Session Planner генерирует пре-сессионный бриф, сканируя файлы проекта (контекст, логи, задачи) и создавая минимальный структурированный план: текущий статус, рекомендуемая цель сессии и приоритизированный список задач. Используйте этот скил в самом начале Cowork-сессии, при возвращении к проекту после перерыва или когда нет чёткого понимания, с чего начать. В отличие от скилов завершения сессии, этот скил помогает войти в сессию, а не выйти из неё.

---

## Требования

- Воркспейс Cowork хотя бы с одним из следующих файлов:
  - `context.md` или `CLAUDE.md` — обзор проекта
  - `logs/log.md` — лог последних действий
  - `TASKS.md` или аналогичный файл задач
- Внешние инструменты и интеграции не нужны
- Работает с любой структурой папок; чем больше файлов — тем детальнее бриф

**Примечание:** Если ни одного из перечисленных файлов нет, скил попросит указать цель сессии вручную.

---

## Как использовать

1. **Откройте Cowork-сессию**
   - Убедитесь, что папка проекта выбрана (или укажите путь)

2. **Запустите скил в начале сессии**
   - Напишите: «бриф сессии» или «подготовь сессию»
   - Опционально добавьте цель — например: «бриф сессии — хочу закончить спек сегодня»

3. **Получите бриф**
   - Скил сканирует файлы проекта и формирует бриф: статус проекта, цель сессии и план работы (до 5 пунктов)
   - Блокеры и отсутствующие файлы фиксируются автоматически

4. **Приступайте к работе**
   - Используйте план как повестку сессии
   - Напишите «сохрани», если хотите сохранить бриф в `output/session-brief.md`

## Примеры

### Пример 1: Начало нового рабочего дня на проекте разработки скилов

**Запуск:** «бриф сессии»

**Результат:**
```
## Сессионный бриф
**Проект:** skills-dev
**Статус:** team-update-aggregator завершён 2026-04-24; cowork-session-planner следующий в бэклоге (RICE 4.0, статус: новый).

### Фокус сессии
Разработать пакет скила cowork-session-planner: спек → пакет → eval → обновление README.

### План работы
1. Написать skill-spec.md для cowork-session-planner
2. Сгенерировать пакет скила (SKILL.md, README, USER-GUIDE)
3. Запустить eval тестирование
4. Обновить корневой README

### Блокеры
- Нет

### Заметки
- Файл задач не найден; план работы составлен на основе backlog.md и log.md
```

---

### Пример 2: Возвращение после трёхдневного перерыва с указанной целью

**Запуск:** «подготовь сессию — нужно закончить eval-отчёт по cowork-session-planner»

**Результат:**
```
## Сессионный бриф
**Проект:** skills-dev
**Статус:** пакет cowork-session-planner сгенерирован 2026-04-27; eval ещё не запускался.

### Фокус сессии
Завершить eval-отчёт для cowork-session-planner и обновить корневой README.

### План работы
1. Запустить eval-runner на пакете cowork-session-planner
2. Проверить eval-report.md и внести правки
3. Обновить ROOT README.md и README.ru.md
4. Обновить статус в backlog.md → готов

### Блокеры
- Нет

### Заметки
- Цель, указанная пользователем, имеет приоритет над автоматически составленным планом.
```

## Триггеры

Используйте любую из этих фраз для запуска скила:

| Русский | English |
|---------|---------|
| бриф сессии | session brief |
| подготовь сессию | prepare my session |
| что делать в этой сессии | what should I work on today in cowork |
| подготовь контекст перед стартом | start my cowork session |
