# Meeting Prep Briefer — Руководство пользователя

Практические сценарии для эффективного использования Meeting Prep Briefer.

---

## Быстрый старт

1. Вставьте расписание или укажите файл: «бриф по встречам — 09:00 Product review (Аня, Миша), 15:00 Статус для стейкхолдеров»
2. Получите одну секцию на каждую встречу с контекстом, открытыми вопросами и повесткой
3. Напишите «сохрани» — скил запишет бриф в `output/meeting-brief-YYYY-MM-DD.md`

---

## Сценарии

### Сценарий 1: Насыщенное утро, несколько встреч подряд

**Ситуация:** У вас 4 встречи с 9:00. На подготовку — 10 минут.

**Что делать:**
1. Откройте Cowork-сессию с выбранной папкой проекта
2. Напишите: «бриф по встречам — [вставьте расписание дня]»
3. Скил просканирует заметки и сформирует бриф по каждой встрече меньше чем за минуту

**Ожидаемый результат:** Один документ с 4 секциями — по одной на встречу — с контекстом из файлов воркспейса. Для встреч без подходящих заметок будет показан промпт-подсказка.

**Совет:** Держите файл `today.md` в воркспейсе и обновляйте его каждое утро — тогда достаточно писать «бриф по встречам из `today.md`» каждый день.

---

### Сценарий 2: Подготовка к важному 1-on-1

**Ситуация:** У вас 1-on-1 с подчинённым. Есть заметки прошлых встреч и файл с темами.

**Что делать:**
1. Убедитесь, что `bob-notes.md` или `1on1-bob.md` есть в воркспейсе с предыдущими темами и action items
2. Напишите: «подготовь встречи — 14:00 1-on-1 с Бобом»
3. Скил использует «боб» как ключевое слово и извлекает контекст из файлов с заметками

**Ожидаемый результат:** Бриф с открытыми action items из прошлой сессии, нерешёнными вопросами и предлагаемой повесткой по блокерам и развитию.

**Совет:** Называйте файлы заметок по именам участников (например, `bob.md`, `anna-notes.md`) — скил использует имена файлов для автоматического подбора контекста.

---

### Сценарий 3: Подготовка встреч на завтра вечером

**Ситуация:** Вы хотите подготовиться к завтрашним встречам перед завершением рабочего дня.

**Что делать:**
1. Сохраните завтрашнее расписание в `tomorrow.md`
2. Напишите: «бриф по встречам из `tomorrow.md` на 2026-04-29»
3. Бриф будет датирован завтрашним числом и сохранён соответственно

**Ожидаемый результат:** Полный бриф на день с завтрашней датой в заголовке, сохранённый в `output/meeting-brief-2026-04-29.md`.

---

## Советы

- **Имена файлов важны:** Называйте контекстные файлы по темам встреч или именам участников (`sprint-review.md`, `anna.md`, `q2-planning.md`) — скил использует их для автоматического подбора контекста.
- **Добавляйте открытые вопросы в заметки:** Если в файлах есть строки с «?» или «открытый вопрос:», скил перенесёт их в секцию «Открытые вопросы».
- **Гибкий формат расписания:** Подходит любой формат с временем и названием в каждой строке: `10:00 Синк`, `10:00 — Синк`, `10:00 | Синк (Алиса, Боб)`.
- **Сохранение для шеринга:** Используйте команду «сохрани» для экспорта брифа в markdown — поделитесь с участниками или сохраните как запись встречи.
- **Без заметок — всё равно полезно:** Даже без контекстных файлов скил сгенерирует предлагаемую повестку на основе названия встречи и участников.
