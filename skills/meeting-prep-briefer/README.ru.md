# Meeting Prep Briefer

Входите на каждую встречу подготовленными — участники, контекст и открытые вопросы из ваших файлов за секунды.

---

## Обзор

Meeting Prep Briefer формирует структурированный бриф для каждой встречи дня. Передайте расписание (вставьте текстом или укажите файл), и скил просканирует рабочее пространство, чтобы собрать бриф по каждой встрече: кто участвует, контекст из ваших файлов, открытые вопросы и предлагаемая повестка. Без интеграций с календарём, без CRM, без API-ключей — работает исключительно с локальными файлами в вашем воркспейсе Cowork.

В отличие от универсальных инструментов подготовки к встречам, этот скил работает без интернета и не требует настройки. Если у вас есть заметки по проектам, файлы участников или записи прошлых встреч в воркспейсе — этого уже достаточно.

---

## Требования

- Расписание дня — вставленное текстом или сохранённое в локальном файле `.md` / `.txt`
- Опционально: файлы контекста в воркспейсе (заметки по проектам, профили участников, записи предыдущих встреч)
- Внешние инструменты и интеграции не нужны

**Примечание:** Скил формирует полезный бриф даже без контекстных файлов — он укажет, для каких встреч заметок не нашлось, и предложит их добавить.

---

## Как использовать

1. **Подготовьте расписание**
   - Вставьте его прямо в чат или сохраните в файл `today.md` / `schedule.md` в воркспейсе
   - Минимальный формат: `10:00 — Синк команды (Алиса, Боб)` — одна встреча в строке

2. **Запустите скил**
   - Напишите: «бриф по встречам» или «подготовь встречи»
   - Опционально укажите файл: «бриф по встречам из `today.md`»

3. **Получите брифы**
   - По одной секции на каждую встречу: участники, контекст, открытые вопросы, предлагаемая повестка
   - Контекст извлекается автоматически из подходящих файлов воркспейса

4. **Сохраните при необходимости**
   - Напишите «сохрани» — скил запишет полный бриф в `output/meeting-brief-YYYY-MM-DD.md`

## Примеры

### Пример 1: Расписание вставлено текстом

**Запуск:** «бриф по встречам — 10:00 Синк команды (Алиса, Боб, Карол), 14:00 1-on-1 с Бобом»

**Результат:**
```
# Бриф по встречам — 2026-04-28

---

## 10:00 — Синк команды

**Участники:** Алиса, Боб, Карол

**Контекст:**
- Из `project-notes.md`: Спринт 12 заканчивается в пятницу; 3 открытые истории.
- Из `alice-notes.md`: Риск по срокам API отмечен как проблема.

**Открытые вопросы:**
- Зависимость по API решена до пятницы?
- Кто отвечает за передачу дизайна?

**Предлагаемая повестка:**
1. Статус спринта (5 мин)
2. Решение по API-риску (10 мин)
3. Ответственные и следующие шаги (5 мин)

---

## 14:00 — 1-on-1 с Бобом

**Участники:** Боб

**Контекст:**
- Файлы контекста не найдены — добавьте заметки в воркспейс.

**Открытые вопросы:**
*(не указаны — добавьте вопросы перед встречей)*

**Предлагаемая повестка:**
1. Текущие блокеры (10 мин)
2. Карьерное развитие (10 мин)
```

---

### Пример 2: Загрузка расписания из файла

**Запуск:** «подготовь встречи из `today.md`»

Скил читает `today.md` из воркспейса, разбирает все встречи, ищет подходящие контекстные файлы и формирует полный бриф на день.
