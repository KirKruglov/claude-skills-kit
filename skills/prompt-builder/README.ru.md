# Prompt Builder

> Создавай структурированные, многоразовые промпты для Claude через интерактивное интервью из 7 вопросов.

---

## Что это делает

Prompt Builder проводит интерактивное интервью и собирает готовый структурированный промпт на основе описания твоей задачи. Вместо того чтобы писать промпт с нуля, ты отвечаешь на 7 вопросов — а Claude берёт на себя работу по форматированию.

На выходе — чистый Markdown-документ с чётко обозначенными секциями:

- **Роль** — какую роль Claude берёт на себя
- **Контекст** — предпосылки и фон, которые нужны Claude
- **Задача** — конкретный результат, который нужно получить
- **Входные данные** — что будет передаваться Claude
- **Требования к выводу** — формат, объём и стиль результата
- **Ограничения** — что нельзя делать, тон, табу
- **Примеры** — опциональная пара input/output для ясности

---

## Как это работает

1. Запускаешь скилл фразой вроде "создай промпт для X"
2. Claude задаёт 7 структурированных вопросов по одному
3. Ты отвечаешь — нечёткие ответы приветствуются, Claude интерпретирует смысл
4. Если ответ неясен, Claude задаёт уточняющий вопрос перед тем как двигаться дальше
5. После всех 7 вопросов Claude собирает и выводит финальный промпт
6. Можно запросить правки или сохранить промпт как `.md` файл

## Быстрый старт

Открой разговор с Claude и напиши:

- "Создай промпт для код-ревью"
- "Напиши промпт для написания статей в блог"
- "Prompt builder — нужен промпт для анализа данных"
- "Помоги собрать промпт для саммаризации встреч"

## Результат

Самостоятельный Markdown-документ, готовый к вставке в любой новый разговор с Claude. Пример структуры:

```markdown
## Роль
Опытный ревьюер кода со специализацией в Python и принципах чистого кода.

## Контекст
Кодовая база — REST API на Django, поддерживается небольшой командой. Следуем PEP 8, предпочитаем явное неявному.

## Задача
Проверить предоставленный код на баги, проблемы со стилем и возможности оптимизации. Приоритет: корректность, затем читаемость.

## Входные данные
Python-сниппет, вставленный напрямую в чат.

## Требования к выводу
Структурированный список из трёх секций: Баги, Проблемы стиля, Предложения по оптимизации. Каждый пункт — краткое объяснение и предлагаемое исправление.

## Ограничения
Не переписывать функцию целиком. Только точечные, минимальные изменения. Технический, прямой тон.
```

---

## Требования

- Аккаунт Claude (бесплатный или платный)
- Загрузка файлов и интеграции не требуются
- Работает в любом интерфейсе: Claude.ai, API, Cowork

---

## Что этот скилл не делает

- Не выполняет и не тестирует сгенерированные промпты — только создаёт их
- Не подключается к внешним инструментам, API или файлам
- Не запоминает ранее созданные промпты между сессиями
- Не генерирует промпты автоматически без интервью — поэтапный процесс это осознанное решение
